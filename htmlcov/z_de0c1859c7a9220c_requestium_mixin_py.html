46 

47 Selenium needs the driver to be currently at the domain of the cookie 

48 before allowing you to add it, so we need to get through this limitation. 

49 

50 The cookie parameter is a dict which must contain the keys (name, value, domain) and 

51 may contain the keys (path, expiry). 

52 

53 We first check that we aren't currently in the cookie's domain, if we aren't, we GET 

54 the cookie's domain and then add the cookies to the driver. 

55 

56 We can override the cookie's domain using 'override_domain'. The use for this 

57 parameter is that sometimes GETting the cookie's domain redirects you to a different 

58 sub domain, and therefore adding the cookie fails. So sometimes the user may 

59 need to override the cookie's domain to a less strict one, Eg.: 'site.com' instead of 

60 'home.site.com', in this way even if the site redirects us to a subdomain, the cookie will 

61 stick. If you set the domain to '', the cookie gets added with whatever domain the browser 

62 is currently at (at least in chrome it does), so this ensures the cookie gets added. 

63 

64 It also retries adding the cookie with a more permissive domain if it fails in the first 

65 try, and raises an exception if that fails. The standard selenium behaviour in this case 

66 was to not do anything, which was very hard to debug.

219def _ensure_click(self: WebElement) -> None: 

220 """ 

221 Ensure a click gets made, because Selenium can be a bit buggy about clicks. 

222 

223 This method gets added to the selenium element returned in '__ensure_element_by_xpath'. 

224 We should probably add it to more selenium methods, such as all the 'find**' methods though. 

225 

226 I wrote this method out of frustration with chromedriver and its problems with clicking 

227 items that need to be scrolled to in order to be clickable. In '__ensure_element_by_xpath' we 

228 scroll to the item before returning it, but chrome has some problems if it doesn't get some 

229 time to scroll to the item. This method ensures chromes gets enough time to scroll to the item 

230 before clicking it. I tried SEVERAL more 'correct' methods to get around this, but none of them 

231 worked 100% of the time (waiting for the element to be 'clickable' does not work). 

232 """ 

233 # We ensure the element is scrolled into the middle of the viewport to ensure that 

234 # it is clickable. There are two main ways an element may not be clickable: 

235 # - It is outside of the viewport 

236 # - It is under a banner or toolbar 

237 # This script solves both cases 

238 script = ( 

239 "var viewPortHeight = Math.max(" 

240 "document.documentElement.clientHeight, window.innerHeight || 0);" 

241 "var elementTop = arguments[0].getBoundingClientRect().top;" 

242 "window.scrollBy(0, elementTop-(viewPortHeight/2));" 

243 ) 

244 self.parent.execute_script(script, self) # parent = the webdriver 

245 

246 exception_message = "" 

247 for _ in range(10): 

248 try: 

249 self.click() 

250 return 

251 except WebDriverException as e: 

252 exception_message = str(e) 

253 time.sleep(0.2) 

254 msg = f"Couldn't click item after trying 10 times, got error message: \n{exception_message}" 

255 raise WebDriverException(msg)
